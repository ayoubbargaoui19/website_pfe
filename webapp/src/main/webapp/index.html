STANDARD SHARING SOFTWARE DEVOPS PROJECT

Ce site web est dans le cadre d'un projet de fin d'etudes à 3S(Standard Sharing Software) dans le sujet : Conception et mise en place d'une plateforme CI/CD pour les applications web multi-services.Ce sujet d'ailleurs implique la notion et la methodologie de travail DEVOPS.En effet, on a arrive a developpe une plateforme et une architecture allant du code jusqu'a le dernier deploiement en passant par Git et Jenkins le moteur de notre plateforme, puis le passage par Ansible qui va designer des roles aux instances contenant Docker et les nodes de Kubernetes et kubernetes master, Ainsi, il y a un autre role qu'il push une image vers DockerHub.

Sonarqube Analyse

During this job, Jenkins will take the code from git and bring it to the sonarqube quality control server, the latter will provide a dashboard on the status of the code in details and the final status according to what we have.

Deploy on Tomcat Server using Jenkins

This job allows us to deploy your application which is in a git directory(GitHub) in a tomcat appache web server. (As another solution that replaces containerization).

Deploy on DockerHost using Jenkins

We are going to create a new job to copy our artifacts in DockerHost.(As soon as they are copied to DockerHost, we can create our own container).
Source file : webapp/target/*.war

Deploy on Container (On DockerHost) using Jenkins

This job is the continuation of job number 3, after the artifacts of the application are in the dockerhost server we will create a docker image and a container linked to this image.

Deploy on Container (On DockerHost) using Ansible and Jenkins

The difference between this job and the previous job is that this one allows us to check if there is another container running and deleting it to create it after an update.

Deploy image on DockerHub and create Container (On DockerHost) using Ansible and Jenkins

A playbook will create an image and deploy it to our dockerhub directory then the dockerhost server will make a pull of this image and create an attached container.

Kubernetes CI JOB

This job will only create a docker image for our application and deploy it to our DockerHub directory. Once this job is successful, there will be automatic execution of job number 8 successively.

Kubernetes CD JOB

This job will make a pull of our docker image in dockerhub and will then execute a container in our kubernetes nodes.
This job can run automatically after job 7.

Deploy artifacts to Nexus

Instead of each time going to bring the code from Github, we can make a copy in Nexus and only make the modification made.
This job allows us to save more time in deployment.

Dockerfile

A Dockerfile is a text document that contains all the commands a user could call on the command line to assemble an image Docker.
This file exists in these servers: DockerHost, Kubernetes Master, Ansible.

Create Devops Image Docker (Playbook for DockerHost)

This playbook will create a Docker image, tag it, and push it on to our DockerHub directory, and delete it from Ansible Server to save space...
This playbook acts only on the docker host server.

Create Devops Project Docker (Playbook for DockerHost)

The role of this playbook is to: stop if we have old docker container, remove stopped docker container, remove current docker image too, pull docker image from dockerhub directory, create container.

Create Devops Image Docker (Playbook for Ansible Server)

This playbook will create a Docker image, tag it, and push it on to our DockerHub directory, and delete it from Ansible Server to save space... This playbook acts only on the Ansible server (See Hosts).

Create deployment
(YAML for Kubernetes Nodes)

This YAML file will deploy our application in the kubernetes nodes and from my DockerHub directory: ayoubbargaoui.
Location : Kubernetes Master
Will be used by playbook number 6.

Execute deployment
(Playbook to execute the previous YAML)

This Ansible playbook will execute the YAML file located in Kubernetes Master and update deployment with new pods if image updated in docker hub.

Create service
(YAML for Kubernetes Nodes)

This YAML file will create a service for users of our application in the kubernetes nodes on port 8080.
Location : Kubernetes Master
Will be used by playbook number 8.

Execute deployment
(Playbook to execute the previous YAML)

This Ansible playbook will execute the previous YAML file located in Kubernetes Master to create a service.

Hosts file

A file that almost all computers and operating systems can use to map a connection between an IP address and domain names.
Location : Ansible server (/opt/kubernetes and /opt/docker)
